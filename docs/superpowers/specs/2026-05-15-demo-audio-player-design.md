# Demo Audio Player — Design Spec

**Date:** 2026-05-15  
**Status:** Approved

---

## Goal

Add a short demo call recording to the landing page so visitors can hear the AI agent handle a real after-hours booking inquiry before signing up.

---

## Script

Two speakers: **Caller** (male, `echo` voice) and **Agent** (AI, female, `nova` voice).

| Speaker | Line |
|---------|------|
| Caller | Hi, I'm trying to book a whale watching tour for this Saturday — I called earlier but no one picked up. |
| Agent | Hey there! You've reached Island Adventures after hours. I'm the AI assistant — happy to help. Saturday whale watching, got it. How many people will be joining? |
| Caller | Just two of us, me and my wife. |
| Agent | Perfect. Can I grab your name and a good number so the team can confirm your spot first thing tomorrow morning? |
| Caller | Sure — Mike Thompson, 555-234-7890. |
| Agent | Got it, Mike. I've noted two guests for Saturday. Someone will call you back before 9 AM to confirm availability and take payment. Anything else tonight? |
| Caller | No, that's great. Thanks. |
| Agent | Enjoy your evening, Mike. We'll be in touch tomorrow. |

---

## Audio Generation Script

**File:** `scripts/generate-demo-audio.ts`

- Uses OpenAI TTS (`tts-1` model, `pcm` response format — 24kHz, 16-bit, mono)
- Iterates over each script line, calls `openai.audio.speech.create()` with the appropriate voice
- Inserts 600ms of silence (zero-byte PCM) between each line
- Prepends a WAV file header (RIFF, 24000Hz, 16-bit, mono) to the concatenated PCM buffer
- Writes output to `public/demo-call.wav`
- Requires `OPENAI_API_KEY` in environment
- Run once manually: `npx ts-node --project tsconfig.server.json scripts/generate-demo-audio.ts`
- Commit the resulting `public/demo-call.wav` — no runtime dependency

### DemoPlayer component

- `'use client'` — uses `useRef` + `useEffect` on an `<audio>` element
- State: `playing`, `currentTime`, `duration`, `activeLine` (index into transcript)
- Each transcript line has a pre-computed `startTime` (sum of prior line durations + silence gaps, estimated at script-load time using char-count heuristic; exact timing can be refined after generation)
- UI elements:
  - **Speaker chips**: two avatar pills (Caller / AI Agent) — active one highlighted in coral
  - **Play/pause button**: coral circle, icon swaps on state
  - **Progress bar**: thin coral bar, clickable to seek
  - **Elapsed / total time**: `MM:SS` format
  - **Transcript**: scrollable list of lines, active line highlighted with coral left border and bold text, inactive lines dimmed
- Styled with inline classes extending the existing v2 design system (navy/tan/coral palette, Fraunces display font for heading, Inter for transcript)
- No external audio libraries — native HTML5 `<audio>` element

### Transcript timing

Exact `startTime` values per line are not known until the WAV is generated. The component accepts a `TRANSCRIPT` constant array with `{ speaker, text, startTime }` entries. After running the generation script, update the `startTime` values by inspecting the generated audio (or log them from the script).

## Out of Scope

- Multiple demo scenarios
- Auto-play or autostart on scroll
- Mobile-specific transcript collapsing
